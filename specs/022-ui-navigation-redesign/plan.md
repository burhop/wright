# Implementation Plan: UI Navigation and Dashboard Redesign

**Branch**: `022-ui-navigation-redesign` | **Date**: 2026-06-09 | **Spec**: [specs/022-ui-navigation-redesign/spec.md](spec.md)

**Input**: Feature specification from `/specs/022-ui-navigation-redesign/spec.md`

## Summary
Redesign the application's top-level navigation, modernize the landing page dashboard into a high-density grid, add a global application log viewer page with a right-click debug integration (chat drawer) powered by Hermes, and add customizable workspace settings (workspace prompts and Git file-size exclusions).

## Technical Context

**Language/Version**: Python 3.11 / TypeScript (React 19, react-router-dom 7, Vite 8)

**Primary Dependencies**: FastAPI (backend), SQLite (DB), structlog (logging), custom SVG components (icons)

**Storage**: SQLite (`state.db`) for system settings and workspace settings; rotating log files (`apps/api/wright.log`) for structured JSON logging.

**Testing**: Playwright (UI integration/E2E), Vitest (React components), pytest (FastAPI endpoint/database integration)

**Target Platform**: Linux server/local execution (offline-capable)

**Project Type**: Monorepo Web Application

**Performance Goals**: Dashboard render <100ms, page-level sidebar transitions <50ms, log loading and query under 1.5s for 10k logs.

**Constraints**: Compliant with Virtual Mechanical Engineer Constitution (zero-server SQLite database, offline-first execution, strict design tokens, and mandatory `data-testid` attributes).

**Scale/Scope**: 4 major dashboard cards, 1 global log page, 1 global settings page, 6 inner workspace buttons, and 2 new SQLite columns.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **FastAPI strict typing**: YES. All new endpoints will use native Pydantic data schemas.
- **Offline-First Mandate**: YES. All logs and settings are stored locally.
- **Relational Data in SQLite (WAL)**: YES. All settings and configuration values will reside in SQLite.
- **Component Test IDs**: YES. Every new button, link, and input will feature unique `data-testid` attributes (e.g. `data-testid="logs-filter-level"`, `data-testid="workspace-prompt-input"`).
- **Structured JSON Logging**: YES. The log parser handles structured JSON lines generated by `structlog`.
