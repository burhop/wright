# Implementation Plan: Agent Docker Container Setup

**Branch**: `010-agent-docker-setup` | **Date**: 2026-06-04 | **Spec**: [spec.md](file:///home/burhop/repos/wright/specs/010-agent-docker-setup/spec.md)

**Input**: Feature specification from `specs/010-agent-docker-setup/spec.md`

## Summary

Implement a production-ready Docker containerization of the full Wright stack (API, web frontend, and agents) with CI/CD-automated builds via GitHub Actions pushing to Docker Hub. The container uses a lean Ubuntu 24.04 base, 7 named volumes for persistence, environment-variable-injected LLM credentials, a container manifest for agent awareness, and host-side backup/restore scripts. Docker is NOT used during normal development.

## Technical Context

**Language/Version**: Python 3.13 (API/agents), Node.js 22+ (web frontend), Bash (entrypoint/scripts)

**Primary Dependencies**: FastAPI, uvicorn, uv (Python package manager), Vite (web bundler), Docker Engine 24+, Docker Compose v2

**Storage**: SQLite (WAL mode) in `/home/agent/` volume, LanceDB in-process

**Testing**: Bash smoke tests for container validation, GitHub Actions CI for build verification

**Target Platform**: Linux amd64 (Dell GB10), Docker container based on Ubuntu 24.04 LTS

**Project Type**: Full-stack containerized web application with AI agent runtime

**Performance Goals**: Image build < 10min, container healthy < 30s, backup < 5min

**Constraints**: No GPU/CUDA in initial image (deferred). Full outbound network access. No Docker in dev workflow.
