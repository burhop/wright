## User Scenarios & Testing *(mandatory)*

### User Story 1 - Enable FreeCAD MCP Server in Tool Registry (Priority: P1)

The user goes to the Engineering Tool Registry in the web dashboard, sees the "FreeCAD Engineering" option, and clicks "Install" / "Connect". The server starts up, discovers its 46+ tools, and registers them in the database for the active workspace.

**Why this priority**: Core MVP requirement to integrate the freecad-mcp server as a selectable tool option in the platform's tool library.

**Independent Test**: Navigate to `/tool-registry`, locate the "FreeCAD Engineering" card, click "Install", verify that status turns to "active", and that the panel lists the standard FreeCAD tools (e.g. `freecad_status`, `step_to_stl`, `create_shape`).

**Acceptance Scenarios**:

1. **Given** the user is on the `/tool-registry` page, **When** they click "Install" on "FreeCAD Engineering", **Then** the backend marks it as installed, executes the stdio command, queries the tools list, and updates the database status to active.
2. **Given** the FreeCAD server is active, **When** the tools list is queried in the database, **Then** all 46+ tools (including CAD, BIM, CFD, and FEM workbench tools) are available in the tool registry.

---

### User Story 2 - Headless FreeCAD Execution in Workspace Sessions (Priority: P2)

An autonomous engineering agent running in a workspace session invokes FreeCAD tools (like primitives creation or STEP-to-STL mesh conversion) to manipulate files in the workspace local path without needing a graphical display.

**Why this priority**: Enables the core AI mechanical engineering capability, allowing agents to generate and convert CAD geometry programmatically.

**Independent Test**: Run a backend tool call to `step_to_stl` on a sample STEP file in the workspace directory, verifying that it runs FreeCADCmd headlessly and writes the output STL file directly to the workspace.

**Acceptance Scenarios**:

1. **Given** a workspace with FreeCAD tool enabled, **When** the agent runs the `create_shape` tool with `shape_type="box"` and dimensions, **Then** a solid box is created headlessly and exported as an STL mesh file in the workspace root.
2. **Given** a STEP assembly file in the workspace, **When** the agent runs the `step_to_stl` tool, **Then** the file is converted to STL mesh and output to the workspace directory.

---

### Edge Cases

- **FreeCAD not installed/missing path**: If the `FREECAD_PATH` is incorrect or FreeCAD is not installed on the system, the server startup will fail, and the database status must show "error" with a clear message: "FreeCADCmd not found at [path]".
- **No display/Xvfb fallback**: If FreeCAD GUI bridge fails to start due to a missing display server, the server must automatically degrade gracefully to headless subprocess mode using the `freecad.cmd` or `FreeCADCmd` executable.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: The system MUST include "FreeCAD Engineering" (with server_id `freecad-engineering-sandraschi`) as an option in the Tool Registry catalog.
- **FR-002**: The host system (GB10) MUST have FreeCAD 1.1+ installed natively via snap (`sudo snap install freecad`), with `FREECAD_PATH` set to `/snap/bin/freecad.cmd` to support headless execution.
- **FR-003**: The Docker container built from `docker/Dockerfile` MUST include FreeCAD 1.1+ (extracted from the official aarch64 AppImage) and have the `FREECAD_PATH` environment variable pre-configured to point to the headless binary.
- **FR-004**: The FreeCAD MCP server command MUST run using `uvx freecad-mcp` or `uv run` and execute tools by piping Python script inputs to the headless FreeCAD interpreter.
- **FR-005**: All output physical assets generated by FreeCAD tools (e.g. `.stl`, `.step`, `.fcstd` files) MUST be saved directly to the active workspace directory or its subdirectories.
- **FR-006**: The system MUST support hot-reloading the newly discovered FreeCAD tools in active agent session configurations.

### Key Entities

- **FreeCAD MCP Server**: The stdio-type server defined in the database registry running the `freecad-mcp` python gateway.
- **FreeCAD Tool**: The discrete command definitions (e.g. `step_to_stl`, `create_shape`, `freecad_status`) queried from the active FreeCAD server and registered in `mcp_tools`.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: FreeCAD availability status check (`freecad_status` tool) completes in under 500ms and returns `freecad_ok: true`.
- **SC-002**: A STEP file conversion to STL (`step_to_stl`) for a standard 10MB mechanical model completes in under 5 seconds.
- **SC-003**: 100% of generated files are written directly into the user's workspace directory without leaking into temporary paths.

## Assumptions

- The host machine has Snap package manager installed and configured (verified: snap 2.75 is installed on the GB10 host).
- The host and container environments have internet access to fetch packages from PyPI and AppImage binary releases on GitHub.
- Docker containers run on an arm64/aarch64 architecture matching the GB10 host system.
