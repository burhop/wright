# GitHub Repository Metadata Configuration Guide

GitHub repository metadata (description, website, and topics) is critical for project discoverability and search engine optimization (SEO). Because these settings must be configured via the GitHub web interface, this guide provides the recommended values and instructions for the repository owner to apply them.

---

## 1. Description & Website

Configure the **About** section on the main repository page with the following:

* **Description**: 
  ```text
  🔧 Design & engineering agent orchestrator — bringing AI productivity gains to traditional engineering roles.
  ```
  *(Note: This is under the 120-character limit to ensure it displays fully in search results and social previews without truncation).*

* **Website**: Leave blank or link to the project's documentation/landing page when deployed.

---

## 2. Topic Tags (20 slots)

Add the following 20 tags to the **Topics** field to maximize repository indexing under relevant categories:

```text
ai-agent, mechanical-engineering, cad, fea, local-first, multi-agent, docker, fastapi, llm, offline-first, manufacturing, openscad, python, typescript, nvidia-dgx, engineering-tools, mcp, parametric-design, digital-twin, autonomous-agent
```

---

## 3. Step-by-Step Configuration Instructions

1. Navigate to the main page of the repository on GitHub (`https://github.com/burhop/wright`).
2. On the right-hand sidebar, locate the **About** section and click the **Gear Icon** (:gear:) next to it.
3. In the dialog box that appears:
   * Paste the **Description** text into the Description field.
   * Enter the **Website** URL if available.
   * Add each of the **Topics** listed above. Press `Enter` or `,` after typing each topic tag.
4. Click **Save changes** to apply.
