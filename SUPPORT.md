# Getting Support for Wright

Welcome to the Wright community! We are excited to help you, but please keep in mind that Wright is in active development and community support is provided on a best-effort basis.

Before opening a new issue or asking a question, please review the resources below to find the quickest path to an answer.

---

## 1. Documentation & Self-Help

For guides, architectural diagrams, and technical details:
* Browse the local `/docs` directory in this repository (e.g., [docs/community-readiness-roadmap.md](file:///home/burhop/repos/wright/docs/community-readiness-roadmap.md) or [docs/technical_analysis.md](file:///home/burhop/repos/wright/docs/technical_analysis.md)).
* Check the `/specs` directory for detailed specifications of implemented features.

---

## 2. Questions & Discussions (GitHub Discussions)

If you have usage questions, configuration issues, or want to discuss design ideas:
* **Channel**: GitHub Discussions Q&A (or open a thread in the Discussions tab of this repository).
* **When to use**: "How do I do X?", "Is it possible to configure Y?", "What LLMs are supported for FEA?".
* **Why**: This keeps our bug tracker clean and allows other community members to search for and find answers to common questions.

---

## 3. Bug Reports & Feature Requests (GitHub Issues)

If you have confirmed a bug in the code or want to propose a new feature:
* **Channel**: GitHub Issues.
* **Search First**: Please search existing open and closed issues to check if your problem has already been reported.
* **Bug Reports**: When opening a bug report, please provide a clear description, reproduction steps, error logs, and details about your environment (OS, model, etc.).
* **Feature Requests**: Describe the user scenario and why this feature would be valuable to the community.
